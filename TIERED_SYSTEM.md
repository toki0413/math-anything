# Math-Anything 分级处理系统

## 核心思想

根据**系统复杂度**和**数据性质**，为用户提供5个级别的功能选项，让用户按需选择。

## 二、数据性质分类

### 2.1 系统大小分类

| 类别 | 原子数 | 推荐级别 | 理由 |
|------|--------|----------|------|
| **小型** | < 100 | Level 1-2 | 传统方法足够 |
| **中型** | 100-1000 | Level 2-3 | 拓扑分析有用 |
| **大型** | 1000-10000 | Level 3-4 | 约束降维显著 |
| **超大型** | > 10000 | Level 4-5 | 潜空间必需 |

### 2.2 时间尺度分类

| 类别 | 模拟时长 | 推荐级别 | 关键需求 |
|------|----------|----------|----------|
| **短时** | < 1 ns | Level 1-2 | 快速初筛 |
| **中时** | 1-100 ns | Level 2-3 | 稳定性重要 |
| **长时** | 100 ns - 1 μs | Level 3-4 | 辛积分器必需 |
| **超长** | > 1 μs | Level 4-5 | 潜空间加速必需 |

### 2.3 约束情况分类

| 类别 | 约束程度 | 推荐级别 | 关键特征 |
|------|----------|----------|----------|
| **无约束** | 柔性系统 | Level 1-3 | 无需流形方法 |
| **部分约束** | 部分刚性 | Level 2-4 | 纤维丛结构 |
| **完全约束** | 刚性系统 | Level 4-5 | SO(3)流形 |
| **周期性** | 晶体/界面 | Level 3-5 | 环面拓扑 |

### 2.4 数据可用性分类

|------|----------|----------|------|

## 三、智能推荐系统

### 3.1 决策树

```
用户输入模拟文件
        ↓
    【Step 1】自动分析文件
        ├── 系统大小: N = ?
        ├── 时间尺度: T = ?
        ├── 约束情况: 刚性/柔性 ?
        └── 数据可用性: 有/无训练数据 ?
        ↓
    【Step 2】推荐级别
        ↓
    如果 N < 100 且 T < 1ns:
        → 推荐 Level 1-2
        → 理由: 小规模短时，传统方法足够
        ↓
    如果 N > 1000 且 有约束:
        → 推荐 Level 4
        → 理由: 大系统+约束，流形降维显著
        ↓
    如果 N > 10000 且 有训练数据:
        → 推荐 Level 5
        → 理由: 超大系统必需潜空间加速
        ↓
    如果 研究相变/多稳态:
        → 推荐 Level 3+
        → 理由: 需要拓扑+Morse分析
        ↓
    如果 长时间模拟 (T > 100ns):
        → 推荐 Level 4+
        → 理由: 需要辛积分器保证稳定性
```

### 3.2 推荐算法

```python
class TieredRecommender:
    """分级推荐系统"""
    
    def recommend_tier(self, file_analysis: Dict) -> TierRecommendation:
        """根据文件分析推荐级别"""
        
        n_atoms = file_analysis['num_atoms']
        time_scale = file_analysis['simulation_time']
        has_constraints = file_analysis['has_constraints']
        has_training_data = file_analysis['has_training_data']
        
        # 计算复杂度分数 (0-100)
        complexity_score = 0
        complexity_score += min(n_atoms / 100, 50)  # 系统大小占50分
        complexity_score += min(time_scale / 1e6, 30)  # 时间占30分
        complexity_score += 20 if has_constraints else 0  # 约束占20分
        
        # 确定级别
        if complexity_score < 20:
            recommended_tier = 1
            suitable_tiers = [1, 2]
        elif complexity_score < 40:
            recommended_tier = 2
            suitable_tiers = [1, 2, 3]
        elif complexity_score < 60:
            recommended_tier = 3
            suitable_tiers = [2, 3, 4]
        elif complexity_score < 80:
            recommended_tier = 4
            suitable_tiers = [3, 4, 5] if has_training_data else [3, 4]
        else:
            recommended_tier = 5 if has_training_data else 4
            suitable_tiers = [4, 5] if has_training_data else [4]
        
        # 生成理由
        reasons = self._generate_reasons(
            n_atoms, time_scale, has_constraints, has_training_data
        )
        
        return TierRecommendation(
            recommended=recommended_tier,
            suitable=suitable_tiers,
            reasons=reasons,
            complexity_score=complexity_score,
            estimated_time=self._estimate_time(recommended_tier),
            required_resources=self._required_resources(recommended_tier)
        )
    
    def _generate_reasons(self, n, t, constraints, data) -> List[str]:
        """生成推荐理由"""
        reasons = []
        
        if n > 1000:
            reasons.append(f"系统较大 ({n}原子)，建议使用高级方法")
        if t > 1e8:  # 100 ns
            reasons.append(f"长时间模拟，需要辛积分器保证稳定性")
        if constraints:
            reasons.append(f"存在约束，流形方法可降维并提高效率")
        if data and n > 5000:
            reasons.append(f"有训练数据且系统大，潜空间可提供10⁶x加速")
        
        return reasons
```

## 五、API设计

### 5.1 分级API

```python
from math_anything import MathAnything

# 初始化
ma = MathAnything()

# 方式1: 自动推荐级别
result = ma.analyze("equil.lmp", auto_tier=True)

# 方式2: 指定级别
result = ma.analyze("equil.lmp", tier=4)

# 方式3: 渐进式分析
result_l2 = ma.analyze("equil.lmp", tier=2)
if result_l2.has_warnings:
    result_l4 = ma.analyze("equil.lmp", tier=4)

# 方式4: 条件触发
result = ma.analyze(
    "equil.lmp",
    min_tier=2,
    max_tier=5,
    upgrade_on_warning=True
)
```
